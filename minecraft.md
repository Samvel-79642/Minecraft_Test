# Мое руководство по работе с Markdown

## Заголовки и подзаголовки
 "# Заголовок" - выделение заголовков. Количество символов "#" задает уровень заголовка (поддерживаются 6 уровней).

## Выделение текста
Для того, чтобы текст сделать полужирным, нужно до и после текста добавить "**" или "__": **Полужирное начертание** или __полужирное начертание__.

## Курсивное начертание
Для того, чтобы сделать курсивное начертание, нужно до и после текста добавить "*" или "_": *курсивное начертание* или _курсивное начертание_.

## Выделение текста полужирным и курсивом
Для того, чтобы сделать полужирное курсивное начертание, нужно до и после текста поставить "***": ***полужирное курсивное начертание***.

## Зачеркнутый текст
Для того, чтобы зачеркнуть текст, нужно до и после текста разместить символ "~~": ~~Зачёркнутый текст~~. 

## Подчеркнутый текст
Для того, чтобы подчеркнуть текст, нужно выполнить следующие действия. В начала слова или текста ставим знак меньше и больше "<>" и в нутри знаков пишем будку "u". Для того, чтобы завершить подчеркивание, в конце слова или текста ставим знак меньше и больше "<>" и внутри знаков пишем "/u", например: <u>подчеркнутый текст</u> 

## Ненумерованный список
Для того, чтобы сделать ненумерованные списки, нужно разместить симов "*" в начале строки:
* Первый
* Второй
* Третий
+ Четвертый
+ Пятый

## Нумерованный список
Для того чтобы сделать нумерованный списов, нужно размещать цифры "1, 2, 3" в начале строки: Данная инструкция написана нумерованным списком, например:
1. Первый
2. Второй
3. Третий

## Табуляция
Для того, чтобы сделать табуляцию или красную строку, нужно в начале строки поставить символ "&" и прописать без пробела "nbsp;", так мы сделаем одинарную табуляцию. Если прописать "ensp;", то мы сделаем двойню табуляцию, например:

&nbsp; Сейчас мы сделали одинарную табуляцию.

&ensp; Сейчас мы сделали двоЙную табуляцию одной командой, аналогично, можно прописать первую команду дважды. 

## Добавление цитат
Для того, чтобы добавить цитату, нужно вначале строки поставить знак больше ">" и после знака написать цитату, например:

> Тут великая цитата.

## Работа с изображением
Для того, чтобы разместить изображение, нужно поставить восклицательный знак "!", поставить квадратные скобки "[имя_фотографии]", в которых будет указано имя фото, без пробела ставим круглые скобки, внутри круглых скобок ставим знаки меньше - больше, между знаками пишем путь к фото (<путь к фото>). Если не ставить восклицательный знак, то мы получим ссылку на фото, например:

![фото_1](<1.jpg>)

Ссылка на фото:

[Фото_1](<1.jpg>)

# Инструкция по работе с GIT

Первый запуск Git.

Создаем на рабочем столе папку. Заходим в VS Code и открываем только что созданную папку. Создаем новый файл, который должен будет отслеживать git, например "<Git_Start.md>".

Далее открываем терминал и проверем работает ли git. Чтобы это сделать, нужно ввести команду:

* git version - проверка версии и работоспособности программы

Если git написал свою версию, значит он готов к работе, если же произошла ошибка или ничего не вывелось, перезапускаем программу, если вновь ничего не происходит, то перезагружаем компьютер.

После проверки работоспособности программы переходим к настройки git, для этого нужно указать имя пользователя и e-mail пользователя. Вводим следующие команды:

* git config --global user.name "<Ваше имя>" 
* git config --global user.email "<адрес_почты@email.com>"

Git готов к работе. 

Далее, чтобы git начал отслеживать изменения в папке, её нужно инициализировать, для этого прописываем в терминале следующую команду:

* git init - инициализация: указываем папку, в которой git начнёт отслеживать изменения

## Основные команды git:

* git status - показывает текущее состояние гита, есть ли изменения, которые нужно закоммитить (сохранить). Если в терминале файл горит красным, значит он не сохранен, если зеленым, то файл сохранен.

* git add _название файла расширения_ - добавления файла для сохрарения

* git commit -m "_название сахранения_" - зафиксировать или сохранить файл

* git log - просмотр журнала изменений

* git log -- graph - визуализация журнала изменений (коммитов)

* git checkout "<ветка на которую нужно переключиться>" — переключение между ветками

* git diff — показывает разницу между текущим файлом и сохранённым

* git branch название ветки – создание новой вспомогательной ветки (допустим для эксперимента или для основной работы, чтобы в будущем добавить в проект)

* git branch – просмотр существующих веток

* git checkout commit — загрузка нужного коммита (берем первые четыре символя от номера коммита, который нужно загрузить)

* git merge text_formatting – (слияние веток) добавили вспомогательную ветку в основную. Для того, чтобы добавление прошло успешно, для начала нужно перейти на основную ветку «master», а затем произвести слияние веток

* git branch -d text_formatting – флаг «-d» позволяет удалять ветки, в данном случае произойдет удаление ветки «text_formatting»

* git reset --hard HEAD~ – удаляет последний сохраненный коммит

## Файл ".gitignore"

* git сохраняет только файлы формата .txt, для того, чтобы он не выдавал ошибку на файлы с другими расширениями, нужно создать файл “.gitignore” и прописывать в этот файл файлы с другим расширением. Открываем файл «.gitignore» и прописываем файлы “image.jpg”». Далее в терминале прописываем команду «git add .gitignore» и делаем коммит «git commit -m “Добавили gitignore файл”». Теперь при добавлении файлов с расширением не «.txt» git не будет выдавать ошибку

## Работа с GitHub

1. Создать аккаунт на https://GitHub.com

2. Создать локальный репозиторий 

3. "Подружить" ваш локальный и удаленный реппозиторий. GitHub при создании нового репозитория подскажет, как это сделать

4. Отправить (push) ваш локальный репозиторий в кдаленный (на GitHub), при этом вам, возможно, нужно будет авторизироваться на удаленном репозитории

5. Провести изменения "с другого компьютера"

6. Выкачать (pull) актуальное состояние из удаленного репозитрия

## Основные поманды git при работе с удаленным репозиторием

* git clone ссылка репозиторий – для того, чтобы скопировать репозиторий с GitHub себе в папку, нужно на GitHub нажать на “Code” и скопировать html ссылку, далее в терминале прописать команду git clone и вставить скопированную ссылку

* cd имя папки/ – смена папки. Не забываем про слеш

## Добавление репозитория на сервер:

+	git remote add origin ссылка на репозиторий – добавление локального репозитория на GitHub. Origin – имя репозитория

+	git branch -M main – выбираем ветку

+	git push -u origin main – сгружаем локальные файлы на облако

+	'+ в первый раз надо проходить авторизацию, VS Code подскажет, как это сделать. 

* git push – в последующие разы для сохранения файлов в облако достаточно просто ввести команду git push

* git pull – эта команда позволяет скачать все файлы из текущего репозитория и автоматически сделать merge (слияние) с нашей версией

## Предложить человеку свой вариант работы/репозиторий (совместная работа)

1.	Делаем форк – копию (fork) интересующего нас репозитория.

2.	Мы делаем git clone _ссыка_ для нашей версии этого репозитория.

3.	Мы создаем ветку с предлагаемыми изменениями.

4.	Производим все изменения только в этой ветке.

5.	Отправляем эти изменения на свой аккаунт (push)

6.	В окне GitHub появляется возможность отправить pull request
